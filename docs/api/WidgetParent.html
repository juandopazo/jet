Class WidgetParent

A widget extension that makes the current widget contein child widgets

Adds a Widget as a child. If the specified Widget already has a parent it will be removed from its current parent before being added as a child
Parameters:
child <object> The Widget instance, or configuration object for the Widget to be added as a child
index <object> (Optional.) Number representing the position at which the child should be inserted

Configuration Attributes

children - object

An array of instances or configuration objects representing the widget's children
Default Value: []

childrenContainer - object

The node inside which to insert the children
Default Value: the content box

childType - object

Constructor reference to the default type of the children managed by this Widget. The default value is taken from defaultChildType

defaultChildType - object

default value used when a childType is not provided. This attribute is used mostly by classes using WidgetParent.
Default Value: WidgetChild

multiple - object

Boolean indicating if multiple children can be selected at once. Whether or not multiple selection is enabled is always delegated to the value of the multiple attribute of the root widget in the object hierarchy
Default Value: false

selectedIndex - Number

The index of the currently selected item

selection - Widget|Array

Returns the currently selected child Widget. If the mulitple attribte is set to true will return an Y.ArrayList instance containing the currently selected children. If no children are selected, will return null
Default Value: null
