responseSchema

Hash

Inherited from DataSource: build\datasource.js:212

The schema by which to parse the response data. May be:

DataSource.responseType.JSARRAY schema A JSARRAY response type assumes the following response shape:

//All records are listed as an array [     ['value1', 'value2', 'value3'], //one record     ['another1', 'another2', 'another3'] //another record ]

DataSource.responseType.TEXT schema This schema essentially splits the string and then acts as if it where a DataSource.responseType.JSARRAY. Must define a fieldDelim property. Example (comma separated value):

responseSchema: {     fieldDelim: ","     fields: ["firstField", "secondField"] }

DataSource.responseType.JSON schema Example:

responseSchema = {     resultList : "Response.Results", // String pointer to result data     // Field order doesn't matter and not all data is required to have a field     fields : [         { key: "id" }, // simple location         { key: "obj" }     ] }

DataSource.responseType.XML schema Example:

responseSchema: {     resultNode: "rootNode", // every result field will be looked for between this node's children     fields: [         { key: "keyName", node: "nodeName" }, // in this case, the value of the field will be the node's value        { key: "otherKey", node: "otherNode", attr: "type" }, // in this other case, the field value will be the "type" attribute         { key: "aNumber", node: "someNode", parser: "parseFloat" } // this value will be parsed as a Float     ] }
