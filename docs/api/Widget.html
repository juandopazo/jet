Class Widget - extends Base

render

void render ( target )
Starts the rendering process. The rendering process is based on custom events. The widget class fires a 'render' event to which all subclasses must subscribe. This way all listeners are fired in the order of the inheritance chain. ie: In the Overlay class, the render process is: render() method -> Module listener -> Overlay listener -> rest of the render() method (appends the boundingBox to the srcNode) This helps to use an easy pattern of OOP CSS: each subclass adds a CSS class name to the boundingBox, for example resulting in
. That way a panel can inherit css properties from module and overlay.
Parameters:
target <NodeList|HTMLElement>
Chainable: This method is chainable.

Configuration Attributes

boundingBox - writeonce NodeList

The bounding box contains all the parts of the widget
Default Value: uses BOUNDING_TEMPLATE instance property

classPrefix - writeonce object

Prefix for all CSS clases. Useful for renaming the project
Default Value: Widget.CSS_PREFIX

contentBox - writeonce NodeList

Another container inside the boundingBox added in order to have a more complex design
Default Value: uses CONTENT_TEMPLATE instance property

height - Number

The height of the overlay. If set to 0 (zero) the height changes with the content

id - object

The id of the widget

rendered - writeonce object

Rendered status. Shouldn't be changed by anything appart from the Widget.render() method
Default Value: false

srcNode - DOMNode | NodeList

The node to which the widget will be appended to. May be set as a configuration attribute, with a setter or as the first parameter of the render() method

width - Number

The width of the overlay
